# Repository development roadmap

## Phase 1 — Public technical scaffold

- Define repository name, scope, and structure.
- Add source publications and publication references.
- Preserve original simulation files.
- Add a cleaned parameterized model entry point.
- Add licensing and citation placeholders.

## Phase 2 — Reproducibility cleanup

- Resolve temperature-unit basis.
- Verify species naming and output mass-flow basis.
- Add baseline simulation output.
- Export CSV results.
- Generate first reproducible biomass/biochar figure.

## Phase 3 — Sensitivity modules

- Temperature sensitivity.
- Moisture sensitivity.
- O2/N2 atmosphere sensitivity.
- Syngas species production trends: CO, H2, CH4, CO2.

## Phase 4 — Carbon-management screening

- Add three-case carbon-management comparison.
- Separate direct process emissions from wider boundary emissions.
- Add transparent assumptions for syngas use, heat supply, and carbon retention.

## Phase 5 — Engineering documentation

- Add feedstock-envelope notes.
- Add scale-up considerations.
- Add validation plan.
- Add product-intent design notes for biochar, heat, syngas, and activated-carbon precursor pathways.

## Phase 6 — Public communication

- Add polished notebooks.
- Add figures for README.
- Add GitHub profile integration.
- Prepare first tagged release.
