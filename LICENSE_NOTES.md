# License notes

The uploaded source publications indicate **CC BY 4.0** in their document headers. For a GitHub repository that contains both documentation and executable code, a practical licensing structure is usually:

- Documentation, figures, and technical notes: **Creative Commons Attribution 4.0 International (CC BY 4.0)**
- Source code: **MIT License** or **BSD-3-Clause License**

Before public release, select the final license strategy and replace this file with formal license files, for example:

```text
LICENSE-CODE.md      # MIT or BSD-3-Clause
LICENSE-DOCS.md      # CC BY 4.0
```

Do not publish the repository with ambiguous licensing if broad reuse is desired.
